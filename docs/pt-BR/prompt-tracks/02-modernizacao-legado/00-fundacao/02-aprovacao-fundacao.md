## Aprenda por Exemplos

### ❌ Exemplo Ruim

```markdown
# 02 - Aprovação: Fundação

## Checklist

- [x] Rodamos o sistema e vimos a visão.

## Definição de Prontidão

- [x] O time está pronto.
```

> **Razão**: Vago. Estar "pronto" sem acesso aos servidores ou orçamento aprovado é um risco de atraso imediato.

### ✅ Exemplo Bom

```markdown
# 02 - Aprovação: Fundação

## Aprovação dos Stakeholders

- **CTO**: Marcos Barbeiro (Aprovado em 14/05/2026)

## Definição de Prontidão

- [x] VPN e permissão de leitura concedidas aos 3 repositórios alvo.
- [x] Alinhamento com o time de suporte sobre a possível flutuação de erros durante a análise.
```

> **Razão**: Define evidências concretas de acesso e comunicação com outros departamentos que podem ser impactados.
